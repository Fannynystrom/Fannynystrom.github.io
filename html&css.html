Html & CSS

HTML och CSS kan uppvelvas som klurigt i början och om du är en av dem som känner så, så kan du vara lugn. Allt inom detta går att googla och hemsidan w3schools kommer vara din bibel. Min lärare sa under en lektion "ni kommer aldrig lära er CSS helt och hållet och om ni gör det så tycker jag synd om er" vilket jag tycker i efterhand sumerar det väldigt bra. Tre stora skillnader att ha med sig: Html gör strukturen, css gör stylingen och Javascript gör funktionen. Bär också alltid dig att Javascript och Java INTE är samma sak. Visual Studio Code är det bästa och enklaste sättet som arbetsverktyg.

Hyper Text Markup Language

När man bygger en hemsida med hjälp av html så kodar man inte, utan man använder ett språk som kallas Markup language. Precis som det låter så markerar man upp sin text i olika boxar som man sedan stylar med css. Element är ett block av innehåll som du använder hela tiden. En rubrik exempelvis är ett block och kan se ut såhär: < h1 > Rubrik < / h1 > och det är viktigt att stänga varje tagg efter önskat innehåll. Struktur Element är de block som märker upp struktur och inte innehåll i det så inte bilder eller rubriker exempelvis. Strukturtaggar är:

  • Header - precis som det låter, innehåller en header.
  • Main - oftast används main runt om flera block, för att kunna styla "main-content" i css.
  • Footer - en fotnot längst ner
  • Nav - är oftast en meny med tillhörande länkänar
  • Section - som main, används ofta för att rama in flera block
  • Article - en artikel på en sida
  • Aside - oftast en pararell artikel vid sidan, kan innehålla text eller meny
  • Div - också en behållare för flera block
Strukturtaggar har inga regler på hur de skall användas, utan de kallas för det för att få struktur, främst för dig själv. När jag började med html använde jag strukturtaggar hela tiden tills jag slutade och började använda div-taggen till alla. Detta för att när du bygger upp sidan och har mer innehåll behöver du ge en "class" till alla strukturtaggar för att css ska veta vart den ska styla. En article exempelvis har man oftast flera stycken av och du måste namnge varje article med en class för att sen i css namne den "article.class(class är det du döper den klassen till". Varför göra det svårt när man kan göra det lätt, använd istället div med en class och class kan vara din strukturtagg. Div är den mest användbara taggen inom webbutveckling.
Lite tips på taggar i HTML:
  • < p > - brödtext, alltså textinehåll. Om du under uppbyggnad inte vet vad du skall ha för textinnehåll men ändå vill se hur det ser ut med text, kan du skriva p och sen lorem för att få random textinnehåll.
  • < b > - gör fetat text (måste märkas efter andra taggen i en kod, så att den har något att göra fetad text på.
  • < i > - kursiv text
  • ul - märker upp en punktlista men du måste använda taggen < li > för varje del du vill ha i punktlistan. Detta som du läser nu är en punktlista.
  • < ol > - samma som ul fast listan är numrerad, här måste du också använda < li > för varje del du vill ha i nummerordning
  • < a > - gör länkar. Du måste använda ett atribut som heter href efter < a h r e f ="här länkar du"> < / l i >
  • < img src = " " alt = " " > bildtagg, där src adresserar till bilden och alt står för bildtexten (när du håller musen över bilden står det vad det är för bild)
  • < h1 > - rubrik, finns upp till < h5 >
  • < br > - gör så att texten bärjar på en ny rad
    • Glöm inte att alla taggar måste avslutas efter önskat innehåll, förutom taggen < br > som inte har en avslutningstagg.

Tips stylingtagg i CSS:
  • line-hight - ökar mellanrummet i textinnehåll så att det blir mer luft emellan. Jag brukar använda någonstans mellan 130%-180% beroende av innehåll.
  • backround - bakrundsfärg, finns också backround-color
  • color - färg för texten i boxen
  • font-family - typsnitt på text
  • font-size - storleken på texten
  • border-radius - rundar ett innehålls kanter så som en bild eller border-box
